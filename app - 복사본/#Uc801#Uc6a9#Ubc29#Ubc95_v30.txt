Sports AI 운영용 API 절약 캐시 v30

핵심 변경
- 사용자가 경기 분석을 열어도 외부 API를 매번 다시 호출하지 않도록 수정
- 동일한 URL의 데이터는 Vercel의 공유 Data Cache에서 재사용
- KBO/MLB/NPB 일정: 5분 캐시
- 최근 경기와 팀 흐름: 10~30분 캐시
- 순위와 시즌 팀 기록: 30분 캐시
- 날씨: 1시간 캐시
- 투수 상세와 구종 자료: 6시간 캐시
- The Odds API 배당: 30분 캐시
- NPB 분석 내부 중복 호출도 캐시 사용

예시
같은 KIA vs 한화 분석을 1,000명이 30분 안에 열어도
동일한 외부 배당 요청은 원칙적으로 캐시된 결과를 사용합니다.
사용자마다 우리 사이트의 내부 API 요청은 발생하지만,
유료 외부 API 크레딧이 사용자 수만큼 차감되는 구조는 아닙니다.

중요
- 이것은 Vercel/Next.js 공유 캐시 방식입니다.
- 완전한 영구 DB 저장 방식은 Supabase 연결 정보가 있어야 별도로 만들 수 있습니다.
- 배당은 경기별/쿼리별 첫 요청과 30분 갱신 시 외부 API를 사용합니다.
- 서로 다른 경기와 서로 다른 날짜는 각각 별도 캐시 항목입니다.

적용 방법
1. sports-ai 프로젝트 최상위에서 압축 해제
2. app 폴더 전체 덮어쓰기
3. 개발 서버 종료
4. PowerShell:
   Remove-Item .next -Recurse -Force
5. npm run dev
6. 브라우저 Ctrl + F5
